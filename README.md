# AstarPathfinding

一个使用A*寻路算法的交互式可视化演示项目，用Pygame实现路径寻找的动画展示。该项目基于ChatGPT-4o实现。

## 项目介绍

本项目是一个A*寻路算法的实现和可视化工具。通过直观的图形界面，展示A*算法如何在二维网格地图中寻找从起点到终点的最优路径。

**主要特性：**
- ✨ 实时动画展示A*算法的寻路过程
- 🎮 支持自定义地图配置
- 🎨 彩色可视化显示探索过程和最终路径
- ⚙️ 可配置的演示速度
- 🔧 简单易用的配置文件格式

**算法说明：**
A*算法是一种启发式搜索算法，结合了Dijkstra算法的精确性和贪心搜索的速度，通过评估函数 `f(n) = g(n) + h(n)` 来找到最优路径。其中：
- `g(n)` 是从起点到当前节点的实际成本
- `h(n)` 是从当前节点到终点的估计成本（曼哈顿距离）

## 使用方法

### 1. 准备配置文件

在项目目录下创建或编辑 `config.txt` 文件：

```
100
oaoxooxoxobo
oxooxoxxooxo
oxooooxooxxo
xoxoxooxooox
oooxoooxxooo
xoooooxoooxo
xoxoxxoxoxoo
oxoooooxoxox
ooxoxoxxoxox
oxooxooxooox
oooooxooooxo
```

**配置说明：**

- **第一行**：演示速度，单位为毫秒（数值越大速度越慢）
- **其他行**：地图数据，必须为正方形网格（行数 = 列数）

**地图符号：**

| 符号 | 含义 |
|------|------|
| `o` | 可通行的空地 |
| `x` | 障碍物 |
| `a` | 起点（绿色） |
| `b` | 终点（红色） |

### 2. 运行程序

```bash
python Astar.py
```

### 3. 界面说明

程序运行后会显示一个可视化窗口，展示A*算法的寻路过程：

- **绿色格子**：已探索的节点
- **蓝色格子**：待探索的节点（开放集）
- **黄色路径**：最终找到的最优路径
- **红色**：起点或终点
- **黑色**：障碍物

关闭窗口即可退出程序。

## 工作原理

A*算法的执行流程：

1. 从起点`a`开始，将其加入开放集
2. 循环执行以下步骤，直到找到终点或无路可走：
   - 从开放集中选择f值最小的节点作为当前节点
   - 检查相邻的四个节点（上下左右）
   - 如果是可通行节点且找到更优路径，则更新其g值和h值
   - 将未访问过的节点加入开放集
   - 标记已探索节点为绿色，待探索节点为蓝色
3. 找到终点后，回溯路径并用黄色标记

## 自定义地图

您可以修改 `config.txt` 文件中的地图数据来创建自己的寻路场景：

- 修改第一行数字改变演示速度
- 修改地图内容改变地图布局
- 确保地图为正方形（行数必须等于列数）
- 必须有且仅有一个起点`a`和一个终点`b`

**示例：**简单的5×5地图

```
200
aoxoo
oxxxo
oooob
xxxxx
ooooo
```

## 常见问题

**Q: 程序报错找不到config.txt怎么办？**

A:  确保config.txt文件与Astar.py在同一目录下。

**Q: 地图必须是正方形吗？**

A: 是的，地图行数和列数必须相等。

**Q: 如何加快或减慢演示速度？**

A: 修改config. txt第一行的数值。数值越小演示越快，数值越大演示越慢。

**Q: 程序找不到路径怎么办？**

A: 检查地图中是否真的存在从起点到终点的通行路线。如果路径被障碍物完全堵住，算法无法找到解决方案。
